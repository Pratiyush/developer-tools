# Regex Tester

## Overview
- **Path:** `/regex-tester`
- **Category:** Development
- **Description:** Test your regular expressions with sample text.
- **Keywords:** regex, tester, sample, expression
- **Redirect from:** None
- **Created:** 2024-09-20
- **Icon:** `Language` (from `@vicons/tabler`)

## Purpose
A live JavaScript regex playground. Users enter a pattern and arbitrary text, and immediately see all non-overlapping matches with their indices, numbered captures, and named groups. Two extras complement the matcher: a **sample-string generator** (RandExp) that produces a string the regex would match, and a **railroad diagram** (Regexper) that visualizes the regex's structure. Six toggle flags (`g`, `i`, `m`, `s`, `u`, `v`) control matching behavior. The pattern persists across reloads.

## Inputs

| Name | Type | Default | Validation |
|------|------|---------|------------|
| `regex` | string | `''` (loaded from query param `regex` or `localStorage` key `regex-tester:regex`) | Must compile via `new RegExp(value)`. Error message: `"Invalid regex: {0}"`. |
| `text` | string | `''` | None. |
| `global` (`g`) | boolean | `true` | — |
| `ignoreCase` (`i`) | boolean | `false` | — |
| `multiline` (`m`) | boolean | `false` | — |
| `dotAll` (`s`) | boolean | `true` | — |
| `unicode` (`u`) | boolean | `true` | — |
| `unicodeSets` (`v`) | boolean | `false` | Only applied when `unicode` is `false` (the two are mutually exclusive in JS — `u` and `v` cannot coexist). |

The `d` flag (capture indices) is **always added** so the matcher can extract `start`/`end` for captures and groups.

## Outputs

### Matches table (one row per match)
| Column | Type | Description |
|--------|------|-------------|
| Index in text | number | `match.index` — character offset of the whole match. |
| Value | string | `match[0]` — the matched substring. |
| Captures | list of `{ name, value, start, end }` | Numbered (positional) capturing groups. `name` is `"1"`, `"2"`, ... |
| Groups | list of `{ name, value, start, end }` | Named groups (`(?<foo>...)`). |

If no matches, an alert "No match" is shown.

### Sample matching text
A string generated by `RandExp` from the regex. Named-group syntax is rewritten to non-capturing because RandExp doesn't support `(?<name>...)`:

```ts
const randexp = new RandExp(new RegExp(regex.value.replace(/\(\?\<[^\>]*\>/g, '(?:')));
return randexp.gen();
```

If the regex is invalid or RandExp can't generate, the sample is `''`.

### Regex Diagram
A railroad-style SVG produced by `@regexper/render` rendered into a shadow DOM (so its inline `<defs>/<style>` doesn't bleed and break the rest of the site's icon transparency).

## UI / Components
The page is `max-w-600px`, three stacked cards:

1. **Regex card (`c-card title="Regex"`)**
   - `c-input-text` multiline (3 rows) with placeholder "Put the regex to test", live validation.
   - `<router-link target="_blank" to="/regex-memo">` — "See Regular Expression Cheatsheet".
   - `n-space` of six checkboxes labeled with their flag letter (`g`, `i`, `m`, `s`, `u`, `v`); each `<span>` has a `title` tooltip explaining the flag.
   - `n-divider`, then a multiline (5 rows) `c-input-text` for the test text.
2. **Matches card (`c-card title="Matches"`)** — `n-table` with header (Index in text, Value, Captures, Groups). Each "Captures" / "Groups" cell is a `<ul>` of `"<name>" = <value> [<start> - <end>]`. Falls back to `c-alert` "No match".
3. **Sample matching text card** — `<pre style="white-space: pre-wrap; word-break: break-all">{{ sample }}</pre>`.
4. **Regex Diagram card** — `<shadow-root ref="visualizerSVG">` with `style="overflow-x: scroll"`. The SVG is mounted via Vue's `vue-shadow-dom` integration.

## Logic / Algorithm

### Flag construction
```ts
let flags = 'd';                           // always include 'd' for indices
if (global)      flags += 'g';
if (ignoreCase)  flags += 'i';
if (multiline)   flags += 'm';
if (dotAll)      flags += 's';
if (unicode)     flags += 'u';
else if (unicodeSets) flags += 'v';        // mutually exclusive with 'u'
```

### `matchRegex(regex, text, flags)` (`regex-tester.service.ts`)
Iterates `re.exec(text)` until exhausted. Loop guards prevent infinite loops on zero-width matches:

```ts
while (match !== null) {
  if (re.lastIndex === lastIndex || match[0] === '') break;
  // collect captures
  Object.entries(match).forEach(([captureName, captureValue]) => {
    if (captureName !== '0' && captureName.match(/\d+/)) {
      captures.push({ name, value, start: indices[Number(name)][0], end: indices[Number(name)][1] });
    }
  });
  // collect named groups
  Object.entries(match.groups || {}).forEach(([groupName, groupValue]) => {
    groups.push({ name, value, start: indices.groups[name][0], end: indices.groups[name][1] });
  });
  results.push({ index: match.index, value: match[0], captures, groups });
  lastIndex = re.lastIndex;
  match = re.exec(text);
}
```

Notes:
- The "captures" are pulled from numeric keys of the match array (`match[1]`, `match[2]`, …) by filtering keys that are pure digits (and not `'0'`, since `match[0]` is the full match).
- Indices come from the `d`-flag-only `match.indices` array (and `match.indices.groups` for named groups).
- The function relies on `re.lastIndex` advancing — it works correctly only when `g` (or `y`) is set; otherwise the loop breaks after the first match because `lastIndex` stays at 0.
- If `re.exec` throws (invalid regex), the call site catches and returns `[]`.

### Sample generation
RandExp is fed a *modified* pattern where named groups are flattened to non-capturing so it doesn't choke on `(?<name>...)`. RandExp doesn't honor flags except case-insensitive — generated samples are best-effort.

### Visualizer effect (`watchEffect`)
1. Get the shadow root of `visualizerSVG`.
2. Remove any existing children.
3. Create a new `<svg>` namespaced element.
4. `await render(regexValue, svg)` from `@regexper/render` — paints into the SVG.
5. Append to the shadow root.

The shadow DOM is required because `@regexper/render` injects a `<defs><style>` block which, in the light DOM, broke icon transparency across the rest of the app.

### Persistence
`useQueryParamOrStorage({ name: 'regex', storageName: 'regex-tester:regex', defaultValue: '' })` synchronizes the `regex` ref with:
1. The `?regex=...` query param if present.
2. Otherwise `localStorage['regex-tester:regex']`.

So a user can share a regex via URL or have it remembered between sessions.

## Edge Cases & Validation
- **Invalid regex** (e.g. unbalanced bracket): validation message "Invalid regex: …" is shown; matches table renders empty (no-match alert), sample is `''`, diagram is cleared.
- **Empty regex**: `new RegExp('')` is valid (matches empty string), so it doesn't fail validation, but the matcher loop's `match[0] === ''` guard breaks immediately, yielding zero matches.
- **Empty text**: yields zero matches.
- **Zero-width match without `g`**: matcher loop breaks after the first iteration to prevent infinite looping.
- **Catastrophic backtracking**: not protected. A pathological regex like `(a+)+$` against a long string can lock the tab; the user's only escape is to change the regex.
- **Named group with same name twice**: Invalid regex, fails validation in JS engines that disallow duplicate group names.
- **`u` and `v` flags together**: prevented in code (`else if`); JS would throw if both were used.
- **Test results** (from `regex-tester.service.test.ts`):
  - `matchRegex('', '', 'd')` → `[]`.
  - `matchRegex('.*', '', 'd')` → `[]`.
  - `matchRegex('', 'aaa', 'd')` → `[]`.
  - `matchRegex('a', 'baaa', 'd')` → one match at index 1, no captures/groups (note: only first match because no `g` flag).
  - `matchRegex('(.)(?<g>r)', 'azertyr', 'gd')` → two matches at indices 2 and 5, each with two numbered captures and a named group `g`.

## Examples
1. **Pattern** `\d{4}` (flags: g, s, u). Text `Phone 1234, ZIP 56789`. Matches: `1234` at 6 (no captures); `5678` at 16. (Trailing `9` left over.)
2. **Pattern** `(?<year>\d{4})-(?<month>\d{2})` (flags: g, i, s, u). Text `2024-09 and 2025-04`. Two matches; each with numbered captures `1`, `2` and named groups `year`, `month` with their start/end indices.

## File Structure
- `index.ts` — Tool registration: hard-coded English name/description (no i18n), keywords, lazy component, `createdAt: 2024-09-20`.
- `regex-tester.vue` — UI, flags, persistence, three cards (matches, sample, diagram), shadow-root SVG injection.
- `regex-tester.service.ts` — Pure `matchRegex(regex, text, flags)` function returning structured results (full match, numbered captures, named groups, all with indices).
- `regex-tester.service.test.ts` — Vitest unit tests for `matchRegex` covering empty inputs, simple matches, and combined positional/named groups.

## Notes
- The `d` flag is mandatory; the matcher relies on `match.indices` to find capture/group offsets.
- `useQueryParamOrStorage` stores just the regex pattern, not the test text or flags. Reloading restores the pattern but starts with empty test text and default flags.
- The link to `/regex-memo` opens in a new tab so the user can reference the cheatsheet without losing their state.
- The shadow DOM workaround for `@regexper/render` is a notable hack — without it, the inserted `<style>` element interferes with icon transparency site-wide.
- Like `regex-memo`, this tool's `index.ts` does not use `translate(...)` — name and description are English-only constants.
